博客搭建笔记

hexo,blog

概述：这是我自己在搭建博客中的的一些步骤详解

搭建个人博客详解

  1. 本贴全程使用Windows+Git环境操作
  2. 在博客部署之前，建议装好Typora（用于编写博客）&Git，以及GitHub的注册
    GitHubd官网
    Typroa官网
    Git官网
    Git的安装操作参考【教程】Git入门安装+使用 by Fing（WHUMSC出品）
  3. 操作Hexo博客配置文件
  4. 常用Hexo部署操作：
    hexo g or hexo generate生成
    hexo s or hexo server本地localhost查看
    hexo dor hexo deploy部署网端
    hexo clean清理缓存
🔴STEP1：Nodejs下载安装（官网地址💨Node.js）

选择左侧的长期支持版

下载完成后直接双击安装，除了配置文件路径以外，其他一路next即可
安装过程中下图的勾勾√记得打（自动安装必要工具）

🔴STEP2：安装包管理工具npm
  • WIN+R打开“运行”窗口，输入cmd，然后CTRL+SHIFT+ENTER以管理员身份运行
  • 输入npm install -g cnpm --registry=http://registry.npm.taobao.org安装淘宝的cnpm（国内镜像速度快）
  • 安装完成后输入cnpm显示npm的相关信息即为安装成功
🔴STEP3：Hexo框架部署与初始化
  • 输入cnpm install -g hexo-cli安装Hexo框架，
  • 安装完之后输入hexo -v显示各项版本信息则视为Hexo框架安装完成
  • 在计算机自己喜欢的任意位置新建blog文件夹用于日后的博客撰写
  • 进入blog文件夹，鼠标右击Git Bash Here，进入Git的操作界面
  • 输入hexo init用于初始化博客，初始化完成后，Hexo会在\blog\source\_posts（博客存储路径）自动生成一篇博文hello-world.md，
  • 输入hexo s即可使用http://localhost:4000/本地访问查看效果(查看此效果要在浏览器中输入对应网址)，查看完毕，要在Git操作界面按CTRL+C`退出这个查看页面，然后执行接下来的操作。
🔴STEP4：Hexo博客新建

此处有两种新建博客的方式：

  1. Git界面输入hexo n “文章名字”，在\blog\source_posts中可以发现新建的博客，cd进入这个路径，输入vim 文章名字.md即可在Git中进行编辑
    个人觉得这个方式比较麻烦，vim操作不方便的同时还可能会出现以下问题：中文乱码

  2. 直接在

    中右击新建文本文档，修改后缀名为.md，使用

    Typora

    （markdown文本御用）编辑即可

    （PS：如果文件未显示后缀，在文件夹查看->勾选文件扩展名

博客新建完成后，在Git中输入hexo clean清理缓存文件

再使用hexo g生成一下刚刚新建的博客，最后hexo s即可在本地localhost预览（http://localhost:4000/）

🔴STEP5：Github远程仓库部署

  • 仓库建立

    进入GitHub新建仓库，用户部署个人博客的github的命名必须符合特定要求，由于我仓库已经建立好所以会报错，点击Create Repository建立完成

!Notice!图中两处的名字需要相同，即YourGithubName.github.io

  • 插件安装与配置
    在Git中输入cnpm install --save hexo-deployer-git在blog目录下安装Git部署插件
    在blog中找到_config.yml文件，右击使用记事本或使用命令 vim +这个文件名（如:vim _config.yml）打开此文件，翻到最下面找到Deployment，更改如下：

    注意type，repo，branch的冒号后均有一个空格！！！ git要有单引号！！！

🔴STEP6：Github博客发布

在Git界面输入hexo -d将博客部署到Github仓库，在部署过程中需要GitHub的账号密码，在没有报错的情况下，部署完成后，通过https://YourGithubName.github.io/则可查看自己的博客

🔴EXTRA STEP：主题更换

Hexo博客的主题参考💨有哪些好看的 Hexo 主题？

  • 将下好的主题文件放到\blog\themes中

  • 或者主题文件在GitHub上面的可以直接克隆到blog文件下的themes文件里（如：git clone https://github.com/litten/hexo-theme-yilia.git theme/yilia ）意思是将这个地址的东西克隆到blog的themes里，并自己生成一个叫这个文件名的文件来存放你从那个地址克隆的所有文件。

  • 配置文件修改（blog文件夹中的_config.yml）（如：vim _config.yml）打开然后修改这个文件

    theme name填写放入themes文件夹中想使用的主题名字

  • 个人信息修改在_config.yml的Site中，除language以外均可自由填写

  • git g 生成一下

  • git s 查看预览一下

  • git d 最后传到远端GitHub上去，就完成了。
